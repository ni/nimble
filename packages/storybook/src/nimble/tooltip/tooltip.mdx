Per [W3C](https://www.w3.org/WAI/ARIA/apg/patterns/tooltip/) - A tooltip is a
popup that displays information related to an element when the element receives
keyboard focus or the mouse hovers over it. It typically appears after a small
delay and disappears when Escape is pressed or on mouse out. It is recommended
to set up aria-describedby, an accessibility feature that sets the description
of another element through ID references. To do this, the anchor element
(button, text, icon, etc.) of the tooltip must have `aria-describedby= name` in
its attributes. To call it, use `id= name` in the nimble-tooltip attributes.
More information can be found in the
[aria-describedby docs](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Attributes/aria-describedby).

## Styling

The tooltip styling can be customized by modyfing the css
properties of `nimble-tooltip::part(control)`. As for example:

```scss
nimble-tooltip::part(control) {
    width: 100px;
    height: 70px;
    overflow: scroll;
}
```

## Usage

The tooltip component is marked as "incubating" because of many limitations
around keyboard and screenreader accessibility that could impact the users.[Nimble limitations](https://github.com/ni/nimble/blob/main/packages/nimble-components/src/tooltip/specs/README.md#open-issues)
The API limitations around anchor ids could result in future breaking API changes.
