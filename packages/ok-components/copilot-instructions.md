# Ok Components - AI Instructions

## Package Context
- **Package**: `@ni/ok-components`
- **Status**: Experimental / Incubating
- **Ownership**: Contributing Team (not Design System Team)
- **Prefix**: `ok-` for custom element tags

## Development Guidelines
- Follows the same patterns as `nimble-components`.
- See `../nimble-components/copilot-instructions.md` for implementation details.
- **Code Quality**: Must adhere to NI and Nimble standards for quality and test coverage.
- **Documentation**: Must use Storybook for documentation.
- **Testing**: Must pass linting and unit tests.

## Code Ownership
- Contributing team owns the code.
- Design System team maintains infrastructure (pipelines, dependencies).
- Contributing team must address build performance issues, test instability, or monorepo burden.
